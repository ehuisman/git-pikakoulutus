Ei "evoluution seuraava askel"

  • SVN – "CVS Done Right"
  • Git – "What Would CVS Not Do?"

Hajautettu

  • Jokainen kopio on tasavertainen
    • Mutta jotkut ovat tasavertaisempia kuin toiset
  • (Lähes) Kaikki operaatiot ovat paikallisia
  • Käytetään silti usein "perinteiseen" tapaan

CVS/SVN

Git

Jokainen versio

  • Täydellinen katsaus sen hetkisestä tilanteesta
  • Viittaukset edeltäjiin
  • Yksilöivä tunniste

Käsitteitä (ja komentoja)

Vaiheet ja tilat

  1. Työhakemisto
  2. Luettelo
  3. Säilö

Työhakemisto (Working Directory)

  • Muutokset edellisen version jälkeen
    1. Ei-seuratut tiedostot status
      • Peruutus: poista tiedostot
    2. Seuratut tiedostot status, diff
      • Muutokset voi peruuttaa checkout -- [tiedosto(t)]

Luettelo (Staging (Area)/Index)

  • Seuraavaan versioon valitut muutokset add, rm
  • Samassa tiedostossa voi olla sekä valittuja että muita muutoksia
    • status, diff --cached
    • add --patch
  • Valinnan voi peruuttaa reset HEAD

Säilö (Repository)

  • Tehdyt muutokset commit
  • Versiohistoria log
  • Historiaa voi muokata
    rebase --interactive, commit --amend

Haarat (Branches)

  • Jokainen kopio on jo oma haaransa
  • Lähtökohtaisesti paikallisia
  • Helppo ja halpa tapa hypätä tekemään jotain ihan muuta
    1. branch
    2. checkout
    • checkout -b
  • Lähtökohtaisesti paikallisia
  • Käytännössä nimi ja osoitin tuoreimpaan versioon

Yhdistäminen (Merge)

  • Haaroilla harvoin tarvetta pitkään ikään
    1. checkout
    2. merge
    3. branch --delete

Rusinat pullasta (Cherry pick)

  • Yksittäisen muutoksen vieminen haarasta toiseen cherry-pick
  • Ei pitäisi olla tarvetta usein

Kätkö (Stash)

  • Väliaikainen paikka muutoksille, jotka haluat säästää myöhempää käyttöä varten stash
  • Ei osa minkään haaran versiohistoriaa
  • Helppo palauttaa stash pop
  • Kätköön mahtuu loputtomasti tavaraa stash list
  • Kätköstä voi myös tehdä haaran stash branch

Etäsäilöt (Remotes)

  • Kun haluaa paikalliset muutokset muidenkin käyttöön
    • push
    • svn dcommit
  • Kun haluaa muiden muutokset paikalliseen säilöön
    • pull/rebase
    • svn rebase
  • Voi olla useita
